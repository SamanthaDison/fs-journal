## Questions

1. What does REST stand for, and in simple terms what does it mean?
REST stands for REpresentational State Transfer; when an API is called, the server will return a represenation of the state of data to the client.

2. What does Stateless mean?
Stateless means the server does not remember anything about the user of the API; the server doesn't remember other GET requests or other requests; each individual request contains all of the information needed to make the request independent of other requests made by the same user.

3. What URL pattern is used when writing a RESTful API?
A URL identifies a resource to be accessed.
