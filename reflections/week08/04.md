## Questions

1. What are the three main types of testing we can accomplish in Vue? What does each method provide?

Unit Testing: allow you to test individual units of code in isolation; you achieve the confidence that as new features are built or your code is refactored your application will remain functional and stable.

Component Testing: requires testing the functionality of Vue components by accessing the DOM; ensures components are mounted and functioning properly.

End to End Testing: validates all layers of the application and invloves testing user interaction/function; includes your frontend code, but all associated backend services and infrastructure that are more representative of the environment that your users will be in.

2. What testing method do you think is the most useful? Why?
I think E2E is the most crucial is the testing process as it involves actually testing and evaluating the user experience.

3. What testing method do you think is the least useful? Why?
I think all testing methods are useful and necessary in their own regards. Each testing method provides the groundwork for the next one. I would say you could perhaps include component testing in the E2E stage.
