## Questions

1. What is a nested route?
Nested routes allow for more dynamic and complex web-pages(especially SPA's). We use nested routes to toggle between pages and components.

2. When might you use a nested route? (other than the provided example)
We may use nested routes when wanted to set up a path or relationship between two components or pages. For example, in today's lecture, you could click on a user's profile image to be directed to their profile page.

3. Can you pass parameters through nested routes? When might you use them?
Yes you can pass through parameters such as id's; this can be used when wanting to access components or data unique to a parent.
