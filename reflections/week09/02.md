## Questions

1. What is a Github action and how do they work?
A GitHub action is a specific action or methods ran when GitHub commands are performed such as push or pull requests.

2. What benefits do Github actions provide?
GitHub actions work to help integrate code as it is being pulled/pushed. This can include comments or tests to verify that a push will still function with the production branch and not break any preexisting code.

3. What types of trigger actions can a workflow use? What do they do?
Workflow: responds to a specific event within the workflow such as a push or pull request
Scheduled: an GitHub action to be performed on a scheduled basis such as a specific time
Webhooks: an action performed by GitHub when accessed through the web such as when somebody stars your repo or when a Wiki page gets created or edited
External: triggered by external event outside of GitHub; repository dispatch event that you would use that when you want something external to communicate back with your GitHub repo
