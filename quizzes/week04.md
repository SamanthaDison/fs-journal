# UnderStanding Asynchronous Code, and API's

**1.** What is the difference between `asynchronous` code and `synchronous` code?
<!-- enter you answer in the space below -->
```
Synchronous code is performed top down and at the same time; asynchronous code 'waits' for certain conditions to be met before running.
```
**2.** What is an event listener?
<!-- enter you answer in the space below -->
```
An event listener 'listens' for certain changes (think ProxyState.on) or conditions to be met to perform.
```
**3.** What does the `O` represent in the `SOLID` principles?
<!-- enter you answer in the space below -->
```
Open-Closed Principle: Objects or entities should be open for extension but closed for modification
```
**4.** What is a callback / higher order function?
<!-- enter you answer in the space below -->
```
Functions that are passed another function as an argument and return a function.
```
**5.** What is a `promise`? How do you capture an error from a `promise`?
<!-- enter you answer in the space below -->
```
A promise is an object within a function that guarantees to do something later; errors are captured by catches.
```
**6.** Name three processes used to make requests over `HTTP`?
<!-- enter you answer in the space below -->
```
GET, PUT, POST, DELETE
```
**7.** What does the `API` acronym stand for?
<!-- enter you answer in the space below -->
```
Aplication Programming Interface
```
**8.** In the `MVC` design pattern, who is responsible for making calls to `APIs`?
<!-- enter you answer in the space below -->

```
Axios and GET requests
```

**9.** What is the purpose of encapsulation in programming?
<!-- enter you answer in the space below -->
```
Encapsulation is used to hide values/functions or the state of data from unauthorized users.
```
**10.** What is `HTTP` response code for a successful request?

```

```
**11.** What is a 500 error?
<!-- enter you answer in the space below -->
```
500 errors indicate a problem with accessing the server that prevents it from fulfilling a request.
```
